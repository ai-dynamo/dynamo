## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-vllm-aggregated">vLLM aggregated</h3>

Single-node TP4 baseline and smallest hardware footprint.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/llama-3-70b.
kubectl apply -f recipes/llama-3-70b/vllm/agg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-vllm-disagg-single-node">vLLM disagg single-node</h3>

P/D split within one node; compare latency against aggregate baseline.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/llama-3-70b.
kubectl apply -f recipes/llama-3-70b/vllm/disagg-single-node/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-vllm-disagg-multi-node">vLLM disagg multi-node</h3>

Two-node disaggregated capacity shape.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/llama-3-70b.
kubectl apply -f recipes/llama-3-70b/vllm/disagg-multi-node/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/llama3-70b-agg-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"RedHatAI/Llama-3.3-70B-Instruct-FP8-dynamic","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>vLLM aggregated</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model RedHatAI/Llama-3.3-70B-Instruct-FP8-dynamic --endpoint-type chat --endpoint /v1/chat/completions --url http://<frontend>:8000 --streaming --synthetic-input-tokens-mean <isl> --output-tokens-mean <osl> --concurrency <from perf.yaml: <concurrency>> --request-count <from perf.yaml> --extra-inputs ignore_eos:true
```
