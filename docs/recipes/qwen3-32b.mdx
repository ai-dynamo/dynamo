## Deploy

<h3 id="deploy-disagg--kv-router">Deploy: Disagg + KV router</h3>

Best candidate when prefix reuse is high enough for KV-aware routing to matter.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-32b`.
kubectl apply -f recipes/qwen3-32b/vllm/disagg-kv-router/deploy.yaml -n ${NAMESPACE}
# Run the benchmark for this variant.
kubectl apply -f recipes/qwen3-32b/vllm/disagg-kv-router/perf.yaml -n ${NAMESPACE}
```

<h3 id="deploy-aggregated-round-robin">Deploy: Aggregated round-robin</h3>

Control run that removes P/D split and KV-aware routing while holding model and traffic constant.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-32b`.
kubectl apply -f recipes/qwen3-32b/vllm/agg-round-robin/deploy.yaml -n ${NAMESPACE}
# Run the benchmark for this variant.
kubectl apply -f recipes/qwen3-32b/vllm/agg-round-robin/perf.yaml -n ${NAMESPACE}
```
