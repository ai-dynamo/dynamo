## When to Use

Use this recipe when requests have long prompts, repeated prefixes or session history, and shorter
generated outputs. The KV router targets TTFT by routing compatible requests to workers with useful
cached blocks; the prefill/decode split targets ITL by reducing decode interruption from long
prefill work. Keep the aggregated round-robin variant as the control run for your own cluster.

## Why This Recipe

This workload has long input sequences, shorter outputs, and meaningful prefix reuse. KV-aware
routing can send requests to workers that already hold matching KV blocks, reducing redundant
prefill. Disaggregated serving separates prefill and decode workers, which helps avoid decode
interruption from long prefill requests.
