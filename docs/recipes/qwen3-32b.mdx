## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-disagg-kv-router">Disagg + KV router</h3>

Best candidate when prefix reuse is high enough for KV-aware routing to matter.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3-32b.
kubectl apply -f recipes/qwen3-32b/vllm/disagg-kv-router/deploy.yaml -n ${NAMESPACE}
```

## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/disagg-router-6p-2d-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"Qwen/Qwen3-32B","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

<h3>Disagg + KV router</h3>
