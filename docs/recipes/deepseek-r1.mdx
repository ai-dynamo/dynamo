## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-sglang-disagg-8-gpu">SGLang disagg 8 GPU</h3>

Single-node SGLang WideEP path with TP=8 per worker; add perf.yaml before making performance claims.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/deepseek-r1.
kubectl apply -f recipes/deepseek-r1/sglang/disagg-8gpu/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-sglang-disagg-16-gpu">SGLang disagg 16 GPU</h3>

Multi-node SGLang WideEP path with TP=16 per worker.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/deepseek-r1.
kubectl apply -f recipes/deepseek-r1/sglang/disagg-16gpu/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-trt-llm-wideep-gb200">TRT-LLM WideEP GB200</h3>

## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/<frontend-service> 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"deepseek-ai/DeepSeek-R1","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash

```
