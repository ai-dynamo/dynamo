## Deploy

<h3 id="deploy-trt-llm-aggregated">Deploy: TRT-LLM aggregated</h3>

Smallest checked-in FP8 deployment footprint; TP2 aggregate.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-32b-fp8`.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/agg/deploy.yaml -n ${NAMESPACE}
# Run the benchmark for this variant.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/agg/perf.yaml -n ${NAMESPACE}
```

<h3 id="deploy-trt-llm-disaggregated">Deploy: TRT-LLM disaggregated</h3>

P/D split for TensorRT-LLM FP8 serving.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-32b-fp8`.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/disagg/deploy.yaml -n ${NAMESPACE}
# Run the benchmark for this variant.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/disagg/perf.yaml -n ${NAMESPACE}
```

<h3 id="deploy-vllm-disaggregated">Deploy: vLLM disaggregated</h3>

vLLM P/D split with NIXL connector; traffic shape differs from TRT rows.

