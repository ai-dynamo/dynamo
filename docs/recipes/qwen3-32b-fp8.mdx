## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-trt-llm-aggregated">TRT-LLM aggregated</h3>

Smallest checked-in FP8 deployment footprint; TP2 aggregate.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3-32b-fp8.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/agg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-trt-llm-disaggregated">TRT-LLM disaggregated</h3>

P/D split for TensorRT-LLM FP8 serving.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3-32b-fp8.
kubectl apply -f recipes/qwen3-32b-fp8/trtllm/disagg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-vllm-disaggregated">vLLM disaggregated</h3>

vLLM P/D split with NIXL connector; traffic shape differs from TRT rows.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3-32b-fp8.
kubectl apply -f recipes/qwen3-32b-fp8/vllm/disagg/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/<frontend-service> 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"Qwen/Qwen3-32B-FP8","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>TRT-LLM aggregated</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model Qwen/Qwen3-32B-FP8 --endpoint-type chat --endpoint /v1/chat/completions --url http://<frontend>:8000 --streaming --synthetic-input-tokens-mean <isl> --output-tokens-mean <osl> --concurrency <from perf.yaml: <concurrency>> --request-count <from perf.yaml> --extra-inputs ignore_eos:true
```
