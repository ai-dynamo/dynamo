## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-dynamo-fd-embedding-cache">Dynamo FD + embedding cache</h3>

Tests frontend decoding plus 8 GiB multimodal embedding cache.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3.6-35b.
kubectl apply -f recipes/qwen3.6-35b/deploy/dynamo-fd-ec.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/${BENCH_FRONTEND} 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"Qwen/Qwen3.6-35B-A3B-FP8","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>Dynamo FD + embedding cache</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model Qwen/Qwen3.6-35B-A3B-FP8 --input-file 30u_8t_5w_8000word_base64.jsonl --custom-dataset-type single_turn --url http://<frontend>:8000 --streaming --request-count 240 --concurrency 30 --extra-inputs max_tokens:1024 --extra-inputs min_tokens:1024 --extra-inputs ignore_eos:true
```

```bash
kubectl apply -f recipes/qwen3.6-35b/perf.yaml -n ${NAMESPACE}
```
