## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-disagg-kv-router-wideep">Disagg + KV router + WideEP</h3>

Tests KV-aware routing, P/D split, and WideEP together on the target trace.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/deepseek-v32-fp4.
kubectl apply -f recipes/deepseek-v32-fp4/trtllm/disagg-kv-router/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash

curl http://localhost:8000/v1/chat/completions \
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>Disagg + KV router + WideEP</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile -m nvidia/DeepSeek-V3.2-NVFP4 --tokenizer nvidia/DeepSeek-V3.2-NVFP4 --input-file <trace>.jsonl --custom-dataset-type mooncake_trace --fixed-schedule --url http://<frontend>:8000 --streaming --goodput "time_to_first_token:20000 inter_token_latency:50"
```

```bash
kubectl apply -f recipes/deepseek-v32-fp4/trtllm/disagg-kv-router/perf.yaml -n ${NAMESPACE}
```
