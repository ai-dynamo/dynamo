## When to Use

Use this recipe when Kimi-K2.5 traffic looks like agentic coding: long context windows, multi-turn
sessions, repeated prefixes, and short decode bursts where both TTFT and decode smoothness matter.
The four TensorRT-LLM variants are designed to isolate the effects of KV-aware routing,
speculative decoding, disaggregation, and host-memory KV offload.

## Results

The disaggregated configuration with KV-aware routing, Eagle decoding, and KV offload is the best
throughput and interactivity point in the benchmarked comparison.

| Variant | tok/s/user avg | tok/s/GPU |
| --- | ---: | ---: |
| Disagg + Eagle + KV router + offload | ~130 | ~5,400 |
| Agg + Eagle + KV router | ~85 | ~4,400 |
| Agg + Eagle + round-robin | ~95 | ~4,000 |
| Agg + round-robin | ~105 | ~1,700 |

<Note>
Treat these as reference metrics, then re-run the provided `perf.yaml` jobs on your own cluster
before using the results for production sizing.
</Note>

## Dataset

The benchmark uses AIPerf's agentic coding trace manifest for 1,000 sessions with an approximately
      200K-token context window. The workload is Mooncake-style, long-context, multi-turn traffic
      with restart splits and layered prefix-cache reuse.

```bash
# Generate the trace from the AIPerf dataset manifest, then copy it into the PVC.
kubectl exec -n ${NAMESPACE} <helper-pod> -- \
  mkdir -p /model-cache/traces/agent_trace_data

kubectl cp <local_trace.jsonl> \
  ${NAMESPACE}/<helper-pod>:/model-cache/traces/agent_trace_data/dataset.jsonl
```

## Configuration Notes

<Accordion title="Recommended TensorRT-LLM path">
The recommended deployment runs the frontend in KV-router mode and separates prefill and decode.
Prefill and decode workers use TP4/EP4. Prefill enables FP8 KV cache block reuse with a 100 GiB
host-memory KV offload tier, while decode uses Eagle speculative decoding against
`nvidia/Kimi-K2.5-Thinking-Eagle3`.
</Accordion>

<Accordion title="Aggregated comparison paths">
The aggregated TensorRT-LLM deployments use three workers with TP8/EP8 across the same 24 GPUs. The
two Eagle variants keep the speculative head and differ only by KV-aware versus round-robin routing.
The no-Eagle baseline keeps round-robin routing and removes speculative decoding.
</Accordion>

<Accordion title="TokenSpeed deploy-only path">
The TokenSpeed recipe is experimental and uses raw Kubernetes `Deployment` and `Service` resources,
not `DynamoGraphDeployment`, because the operator backend enum does not yet admit `tokenspeed`. It
requires a locally built Dynamo+TokenSpeed image and uses explicit `chat_template_kwargs` and
`skip_special_tokens` settings for the Kimi-K2.5 reasoning parser contract.
</Accordion>
