## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-disagg-eagle-kv-router-offload">Disagg + Eagle + KV router + offload</h3>

Best benchmarked point for throughput and interactivity on KV-reuse-heavy traffic.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/kimi-k2.5.
kubectl apply -f recipes/kimi-k2.5/trtllm/disagg-eagle-kv-router/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/kimi-k25-disagg-kv-eagle-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"nvidia/Kimi-K2.5-NVFP4","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>Disagg + Eagle + KV router + offload</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile -m nvidia/Kimi-K2.5-NVFP4 --tokenizer nvidia/Kimi-K2.5-NVFP4 --input-file /model-cache/traces/agent_trace_data/dataset.jsonl --custom-dataset-type mooncake_trace --url http://kimi-k25-disagg-kv-eagle-frontend:8000 --streaming --concurrency 32 --benchmark-duration 3600 --goodput "time_to_first_token:5000 inter_token_latency:10"
```

```bash
kubectl apply -f recipes/kimi-k2.5/trtllm/disagg-eagle-kv-router/perf.yaml -n ${NAMESPACE}
```
