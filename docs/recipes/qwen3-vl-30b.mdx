## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-aggregated-embedding-cache-on">Aggregated + embedding cache ON</h3>

Higher output TPS and lower TTFT when repeated images remain cacheable.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/qwen3-vl-30b.
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/qwen3-vl-agg-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"Qwen/Qwen3-VL-30B-A3B-Instruct-FP8","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>Aggregated + embedding cache ON</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model Qwen/Qwen3-VL-30B-A3B-Instruct-FP8 --input-file qwen3_vl_1000req_1img_pool200.jsonl --custom-dataset-type single_turn --url http://qwen3-vl-agg-frontend:8000 --streaming --request-count 1000 --concurrency 64 --extra-inputs max_tokens:150 --extra-inputs min_tokens:150 --extra-inputs ignore_eos:true
```

```bash
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/perf.yaml -n ${NAMESPACE}
```


## Expected Performance

| Metric | Cache ON | Cache OFF | Delta |
| --- | ---: | ---: | ---: |
| Output TPS | 3,575.6 tok/s | 3,072.3 tok/s | +16.4% |
| TTFT avg | 526.0 ms | 727.5 ms | -27.7% |
| TTFT p50 | 356.8 ms | 510.8 ms | -30.1% |
| ITL avg | 14.1 ms | 15.5 ms | -8.8% |
| Request latency avg | 2,630.0 ms | 3,035.7 ms | -13.4% |
