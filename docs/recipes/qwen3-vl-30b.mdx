## When to Use

Use this page when your deployment decision starts with `Qwen/Qwen3-VL-30B-A3B-Instruct-FP8` and you need to understand which checked-in recipe asset matches your backend, hardware, and workload shape. The variant role column is part of the design: a control row, treatment row, alternative row, and deploy-only row should not be interpreted the same way.

## Results

The source README includes numeric cache-on/cache-off results, so this page treats cache ON as the recommended treatment and cache OFF as the control.

| Metric | Cache ON | Cache OFF | Delta |
| --- | ---: | ---: | ---: |
| Output TPS | 3,575.6 tok/s | 3,072.3 tok/s | +16.4% |
| TTFT avg | 526.0 ms | 727.5 ms | -27.7% |
| TTFT p50 | 356.8 ms | 510.8 ms | -30.1% |
| ITL avg | 14.1 ms | 15.5 ms | -8.8% |
| Request latency avg | 2,630.0 ms | 3,035.7 ms | -13.4% |

## Deploy

<h3 id="deploy-aggregated--embedding-cache-on">Deploy: Aggregated + embedding cache ON</h3>

Higher output TPS and lower TTFT when repeated images remain cacheable.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-vl-30b`.
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/deploy.yaml -n ${NAMESPACE}
# Run the benchmark for this variant.
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/perf.yaml -n ${NAMESPACE}
```

<h3 id="deploy-aggregated-cache-off-baseline">Deploy: Aggregated cache OFF baseline</h3>

Control run for measuring cache impact before changing image pool, cache size, or request rate.

```bash
export NAMESPACE=your-namespace


# Prepare shared storage and model assets using the setup manifests in `recipes/qwen3-vl-30b`.
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/deploy.yaml -n ${NAMESPACE}
# Cache-off comparison arm:
# 1. Set DYN_MULTIMODAL_EMBEDDING_CACHE_GB=0 in the deploy manifest.
# 2. Set CACHE_MODE=cache_off in the perf manifest.
# 3. Apply the same deploy and perf files.
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/deploy.yaml -n ${NAMESPACE}
kubectl apply -f recipes/qwen3-vl-30b/vllm/agg-embedding-cache/perf.yaml -n ${NAMESPACE}
```

## Notes

- Cache OFF is implemented by setting DYN_MULTIMODAL_EMBEDDING_CACHE_GB=0 and CACHE_MODE=cache_off before applying the manifests.
- Exact hit rate still depends on runtime eviction behavior, not just image reuse in the generated JSONL.
