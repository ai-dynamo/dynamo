## When to Use

Use the aggregated recipe as the simpler Blackwell baseline for short prompts and long outputs. Use
the disaggregated recipe when prefill work is large enough that separating prefill and decode is the
serving question you need to answer. Keep the two benchmark shapes separate: the aggregated job is a
128-token input run, while the disaggregated job is an 8K-token input run.

<Warning>
Edit `storageClassName`, namespace, image tags, and secrets before applying these manifests. The
aggregate path requires ARM64 GB200 nodes and will not run on x86 Hopper or Ampere hardware.
Disaggregated startup can take roughly 15-30 minutes depending on storage speed.
</Warning>

<Note>
The disaggregated deploy topology is 5 GPUs: 1 prefill GPU and 4 decode GPUs. Its benchmark YAML
sets `DEPLOYMENT_GPU_COUNT=6`, which is what produces the 1,536 total concurrency value. Until that
is reconciled in the source recipe, treat `DEPLOYMENT_GPU_COUNT` as the benchmark job's concurrency
input rather than the deployment topology.
</Note>

## Configuration Notes

<Accordion title="Aggregated TensorRT-LLM">
The aggregate worker uses TP4/EP4, a round-robin frontend, `TRTLLM_ENABLE_PDL=1`,
`TRT_LLM_DISABLE_LOAD_WEIGHTS_IN_PARALLEL=True`, `max_batch_size=800`, and
`free_gpu_memory_fraction=0.9`. The ConfigMap enables attention DP and uses the CUTLASS MoE backend.
</Accordion>

<Accordion title="Disaggregated prefill/decode">
The disaggregated ConfigMap gives prefill and decode separate engine configs. Prefill uses TP1,
`max_batch_size=64`, `free_gpu_memory_fraction=0.8`, and disables the overlap scheduler. Decode uses
TP4, `max_batch_size=1280`, `free_gpu_memory_fraction=0.85`, and keeps the overlap scheduler enabled.
</Accordion>

<Accordion title="KV transfer">
The disaggregated recipe uses the UCX cache transceiver with `max_tokens_in_buffer=9216` to transfer
KV cache blocks between prefill and decode.
</Accordion>

<Accordion title="Quantization">
The disaggregated deployment sets `OVERRIDE_QUANT_ALGO=W4A8_MXFP4_MXFP8`. Keep this override with
the checked-in engine configs unless you are deliberately testing another quantization path.
</Accordion>
