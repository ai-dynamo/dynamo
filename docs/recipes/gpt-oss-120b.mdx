## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-tensorrt-llm-aggregated">TensorRT-LLM aggregated</h3>

Use for short-prompt, long-output, high-concurrency throughput testing.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/gpt-oss-120b.
kubectl apply -f recipes/gpt-oss-120b/trtllm/agg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-tensorrt-llm-disaggregated">TensorRT-LLM disaggregated</h3>

Use for long-context generation where P/D separation is the target shape.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/gpt-oss-120b.
kubectl apply -f recipes/gpt-oss-120b/trtllm/disagg/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/gpt-oss-agg-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"openai/gpt-oss-120b","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>TensorRT-LLM aggregated</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model openai/gpt-oss-120b --endpoint-type chat --endpoint /v1/chat/completions --url http://<frontend>:8000 --streaming --synthetic-input-tokens-mean <isl> --output-tokens-mean <osl> --concurrency <from perf.yaml: <concurrency>> --request-count <from perf.yaml> --extra-inputs ignore_eos:true
```

```bash
kubectl apply -f recipes/gpt-oss-120b/trtllm/agg/perf.yaml -n ${NAMESPACE}
```
