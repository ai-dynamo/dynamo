## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-vllm-aggregated">vLLM aggregated</h3>

TP4 aggregate path with approximate KV-aware routing and Mamba cache workaround.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/nemotron-3-super-fp8.
kubectl apply -f recipes/nemotron-3-super-fp8/vllm/agg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-tensorrt-llm-disaggregated">TensorRT-LLM disaggregated</h3>

P/D split using UCX transfer because hybrid cache state is not NIXL/Mooncake-safe here.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/nemotron-3-super-fp8.
kubectl apply -f recipes/nemotron-3-super-fp8/trtllm/disagg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-sglang-aggregated">SGLang aggregated</h3>

TP4 aggregate SGLang path; verify against current Dynamo image before claiming support.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/nemotron-3-super-fp8.
kubectl apply -f recipes/nemotron-3-super-fp8/sglang/agg/deploy.yaml -n ${NAMESPACE}
```
<h3 id="deploy-sglang-disaggregated">SGLang disaggregated</h3>

P/D split with nixl transfer in the README; no benchmark coverage today.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/nemotron-3-super-fp8.
kubectl apply -f recipes/nemotron-3-super-fp8/sglang/disagg/deploy.yaml -n ${NAMESPACE}
```
