## Deployment Instructions

<div id="deploy"></div>

<h3 id="deploy-sglang-disaggregated-prefill-decode">SGLang disaggregated prefill/decode</h3>

Use as the current benchmark-backed GLM-5 path; it stresses long output with a large decode rank group.

```bash
export NAMESPACE=your-namespace

# Prepare model cache and model assets using setup manifests in recipes/glm-5-nvfp4.
kubectl apply -f recipes/glm-5-nvfp4/sglang/disagg/deploy.yaml -n ${NAMESPACE}
```


## Smoke Test

<div id="smoke-test"></div>

Run a small request before starting the benchmark. Replace the service name if your deployment exposes a different frontend.

```bash
export NAMESPACE=your-namespace
kubectl port-forward svc/glm5-sglang-frontend 8000:8000 -n ${NAMESPACE}

curl http://localhost:8000/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"zai-org/GLM-5-NVFP4","messages":[{"role":"user","content":"Write a one-sentence readiness check."}],"max_tokens":64}'
```

## Benchmark Instructions

<div id="benchmark"></div>

<h3>SGLang disaggregated prefill/decode</h3>

The checked-in `perf.yaml` wraps this AIPerf run. Use the manifest for the Kubernetes job, and keep this command visible so the traffic shape is understandable.

```bash
aiperf profile --model nvidia/GLM-5-NVFP4 --tokenizer nvidia/GLM-5-NVFP4 --endpoint-type chat --endpoint /v1/chat/completions --url http://glm5-sglang-frontend:8000 --streaming --synthetic-input-tokens-mean 1000 --output-tokens-mean 8192 --concurrency 512 --request-count 1536 --extra-inputs max_tokens:8192 --extra-inputs min_tokens:8192 --extra-inputs ignore_eos:true
```

```bash
kubectl apply -f recipes/glm-5-nvfp4/sglang/disagg/perf.yaml -n ${NAMESPACE}
```
