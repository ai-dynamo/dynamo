## Results

<div id="results"></div>

| Metric | Cache ON | Cache OFF | Delta |
| --- | ---: | ---: | ---: |
| Output TPS | 3,575.6 tok/s | 3,072.3 tok/s | +16.4% |
| TTFT avg | 526.0 ms | 727.5 ms | -27.7% |
| TTFT p50 | 356.8 ms | 510.8 ms | -30.1% |
| ITL avg | 14.1 ms | 15.5 ms | -8.8% |
| Request latency avg | 2,630.0 ms | 3,035.7 ms | -13.4% |

## Reproduce

<div id="reproduce"></div>

The `perf.yaml` files wrap the AIPerf command below. Run each arm in isolation unless the source README says the benchmark can run side by side.

```bash
aiperf profile --model Qwen/Qwen3-VL-30B-A3B-Instruct-FP8 --input-file qwen3_vl_1000req_1img_pool200.jsonl --custom-dataset-type single_turn --url http://qwen3-vl-agg-frontend:8000 --streaming --request-count 1000 --concurrency 64 --extra-inputs max_tokens:150 --extra-inputs min_tokens:150 --extra-inputs ignore_eos:true
```

```bash
export NAMESPACE=your-namespace
# Pick one arm from the table above, deploy it, wait for readiness, then apply its perf.yaml.
kubectl apply -f <arm>/deploy.yaml -n ${NAMESPACE}
kubectl apply -f <arm>/perf.yaml -n ${NAMESPACE}
```
