## Results

<div id="results"></div>

| Arm | tok/s/user avg | tok/s/GPU | Interpretation |
| --- | ---: | ---: | --- |
| Disagg + Eagle + KV routing + offload | ~130 | ~5,400 | Best benchmarked point |
| Agg + Eagle + KV routing | ~85 | ~4,400 | Keeps routing and Eagle |
| Agg + Eagle + round-robin | ~95 | ~4,000 | Isolates routing effect |
| Agg + round-robin | ~105 | ~1,700 | Base control without Eagle |

## Reproduce

<div id="reproduce"></div>

The `perf.yaml` files wrap the AIPerf command below. Run each arm in isolation unless the source README says the benchmark can run side by side.

```bash
aiperf profile -m nvidia/Kimi-K2.5-NVFP4 --tokenizer nvidia/Kimi-K2.5-NVFP4 --input-file /model-cache/traces/agent_trace_data/dataset.jsonl --custom-dataset-type mooncake_trace --url http://<frontend>:8000 --streaming --concurrency <24-or-32> --benchmark-duration 3600 --goodput "time_to_first_token:5000 inter_token_latency:10"
```

```bash
export NAMESPACE=your-namespace
# Pick one arm from the table above, deploy it, wait for readiness, then apply its perf.yaml.
kubectl apply -f <arm>/deploy.yaml -n ${NAMESPACE}
kubectl apply -f <arm>/perf.yaml -n ${NAMESPACE}
```
