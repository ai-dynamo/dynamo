A `DynamoGraphDeployment` (DGD) is the canonical description of a running Dynamo inference graph. You list the components that make up the graph — frontend, workers, prefill and decode workers, planner, EPP — and the [operator](dynamo-operator.md) reconciles them into [`DynamoComponentDeployment`](dcd-reference.mdx) resources, pods, and Services.

A DGD is the resource that persists and serves traffic. Author one directly when you have a known-good configuration or a tuned [recipe](https://github.com/ai-dynamo/dynamo/tree/main/recipes); generate one from intent with a [`DynamoGraphDeploymentRequest`](dgdr-reference.mdx) (DGDR).

This page documents the `nvidia.com/v1beta1` API — the served, current version.

<Info>
  Each entry in `spec.components` is a `DynamoComponentDeploymentSharedSpec` — the same fields documented on the [DCD Reference](dcd-reference.mdx#shared-component-spec). This page covers the **graph-level** fields; per-component fields live there.
</Info>

## Minimal example

```yaml
apiVersion: nvidia.com/v1beta1
kind: DynamoGraphDeployment
metadata:
  name: my-graph
spec:
  backendFramework: vllm
  components:
    - name: frontend
      type: frontend
      replicas: 1
    - name: worker
      type: worker
      replicas: 1
      podTemplate:
        spec:
          containers:
            - name: main
              image: nvcr.io/nvidia/ai-dynamo/vllm-runtime:0.6.0
```

## Spec reference

<ParamField path="components" type="[]DynamoComponentDeploymentSharedSpec">
  Components deployed as part of this graph. Each entry carries its own stable logical `name` (unique within the list, case-insensitively). Component types are repeatable except `type: epp`, which may appear at most once. Maximum 25 components. For the full per-component field set, see [DCD Reference — Shared component spec](dcd-reference.mdx#shared-component-spec).
</ParamField>

<ParamField path="backendFramework" type="'sglang' | 'vllm' | 'trtllm'">
  Backend framework for the graph. Sets backend-specific defaults the operator injects into each component's `main` container.
</ParamField>

<ParamField path="env" type="[]core/v1.EnvVar">
  Environment variables prepended to every component's environment. A component-specific `env` entry with the same name takes precedence and may reference values from this list.
</ParamField>

<ParamField path="annotations" type="map[string]string">
  Annotations propagated to all child resources (scaling adapters, DCDs, Deployments, and pod templates). Component-level `podTemplate` values take precedence on conflict.
</ParamField>

<ParamField path="labels" type="map[string]string">
  Labels propagated to all child resources. Same precedence rules as `annotations`.
</ParamField>

<ParamField path="priorityClassName" type="string">
  Name of the `PriorityClass` to use for Grove `PodCliqueSet`s. Requires the Grove pathway. See [Grove](grove.md).
</ParamField>

<ParamField path="restart" type="object">
  Restart policy for the graph. Change `restart.id` to trigger a restart.
</ParamField>

<Indent>
  <ParamField path="restart.id" type="string" required={true}>
    Arbitrary string; any change to it initiates a restart of the graph according to the configured strategy.
  </ParamField>
  <ParamField path="restart.strategy.type" type="'Sequential' | 'Parallel'" default="Sequential">
    How components are restarted.
  </ParamField>
  <ParamField path="restart.strategy.order" type="[]string">
    Complete ordered set of component names for a sequential restart. Omit to use the controller's default order. Must not be set for parallel restarts.
  </ParamField>
</Indent>

<ParamField path="topologyConstraint" type="object">
  Deployment-level topology constraint. Components without their own `topologyConstraint` inherit this value. See [Topology-Aware Scheduling](topology-aware-scheduling.md).
</ParamField>

<Indent>
  <ParamField path="topologyConstraint.clusterTopologyName" type="string" required={true}>
    Name of the `ClusterTopology` resource defining the topology hierarchy for this deployment.
  </ParamField>
  <ParamField path="topologyConstraint.packDomain" type="string">
    Default topology domain to pack pods within. Optional at this level; omit when only components carry constraints.
  </ParamField>
</Indent>

<ParamField path="experimental" type="object">
  Graph-level opt-in preview features whose API shape may change in breaking ways between `v1beta1` releases. Component-level experimental features live under `spec.components[*].experimental` instead — see [DCD Reference — Experimental fields](dcd-reference.mdx#experimental-fields).
</ParamField>

<Indent>
  <ParamField path="experimental.kvTransferPolicy" type="object">
    Topology-aware routing for KV-cache transfers between prefill and decode workers. Set exactly one of `labelKey` or `clusterTopologyName`. See [Topology-Aware KV Transfer](topology-aware-kv-transfer.md).
  </ParamField>
</Indent>

## Status

The operator maintains observed state under `status`.

<ParamField path="status.state" type="string" default="initializing">
  High-level textual status of the graph deployment lifecycle.
</ParamField>

<ParamField path="status.conditions" type="[]metav1.Condition">
  Latest observed conditions, merged by type. Includes `Available` and `DynamoComponentReady`.
</ParamField>

<ParamField path="status.components" type="map[string]ComponentReplicaStatus">
  Per-component replica status, keyed by component name.
</ParamField>

<ParamField path="status.observedGeneration" type="integer">
  Most recent `metadata.generation` observed by the controller.
</ParamField>

<ParamField path="status.restart" type="object">
  Status of a graph-level restart in progress.
</ParamField>

<ParamField path="status.checkpoints" type="map[string]ComponentCheckpointStatus">
  Per-component checkpoint status, keyed by component name. See [Snapshot and Restore](snapshot.md).
</ParamField>

<ParamField path="status.rollingUpdate" type="object">
  Progress of operator-managed rolling updates. Currently supported only for single-node, non-Grove deployments. See [Rolling Update](rolling-update.md).
</ParamField>

## Inspect a DGD

```bash
# List graph deployments (short name: dgd)
kubectl get dgd -n $NAMESPACE

# Detailed status, conditions, and events
kubectl describe dgd my-graph -n $NAMESPACE

# Per-component readiness
kubectl get dgd my-graph -n $NAMESPACE -o jsonpath='{.status.components}'
```
