A `DynamoComponentDeployment` (DCD) describes one component of a Dynamo inference graph â€” a frontend, a worker, a prefill or decode worker, a planner, or an EPP. The [operator](dynamo-operator.md) reconciles each DCD into a Kubernetes Deployment (or Grove workload) plus its Services, ConfigMaps, and pods.

A DCD is rarely authored on its own. Each entry in a [`DynamoGraphDeployment`](dgd-reference.mdx) `spec.components` list is a `DynamoComponentDeploymentSharedSpec`, the same shape documented here under [Shared component spec](#shared-component-spec). The operator creates one child DCD per component. Author a standalone DCD only when you want to manage a single component's lifecycle independently; otherwise define components inside a DGD.

This page documents the `nvidia.com/v1beta1` API â€” the served, current version.

<Info>
  This reference covers user-configurable spec fields. For operator-injected defaults (images, ports, probes, resources) and environment variables, see the [Dynamo Operator](dynamo-operator.md) guide.
</Info>

## Minimal example

```yaml
apiVersion: nvidia.com/v1beta1
kind: DynamoComponentDeployment
metadata:
  name: my-worker
spec:
  backendFramework: vllm
  type: worker
  replicas: 1
  podTemplate:
    spec:
      containers:
        - name: main
          image: nvcr.io/nvidia/ai-dynamo/vllm-runtime:0.6.0
```

## Spec reference

The DCD spec is `backendFramework` plus the [shared component spec](#shared-component-spec) inlined at the same level.

<ParamField path="backendFramework" type="'sglang' | 'vllm' | 'trtllm'">
  Inference backend framework for this component. Drives backend-specific defaults the operator injects into the `main` container.
</ParamField>

### Shared component spec

These fields are shared between a standalone DCD and each entry of a DGD `spec.components` list. In a DGD, prefix them with `spec.components[*]`.

<ParamField path="name" type="string" required={true}>
  Stable logical identifier for the component, unique within its parent DGD's `spec.components` list. Must match `^[A-Za-z0-9]([-A-Za-z0-9]*[A-Za-z0-9])?$`, 1â€“63 characters. For a standalone DCD the defaulting webhook populates `name` from `metadata.name`, so you rarely set it explicitly. The name is decoupled from the underlying workload name so the operator can rename child workloads (for example, hash-suffixing worker DCDs during a rolling update) without losing the identity that labels, status maps, scaling adapters, planner RBAC, and EPP filters depend on.
</ParamField>

<ParamField path="type" type="'frontend' | 'worker' | 'prefill' | 'decode' | 'planner' | 'epp'">
  Role of the component within the graph. Drives port mapping, frontend detection, planner RBAC, and the pod label `nvidia.com/dynamo-component-type`. `prefill` and `decode` are first-class values for disaggregated serving and can be set directly. At most one component per graph may be `epp`.
</ParamField>

<ParamField path="replicas" type="integer">
  Desired number of pods for this component. Minimum `0`. When `scalingAdapter` is set, this field is owned by the [DynamoGraphDeploymentScalingAdapter](autoscaling.md) and should not be modified directly.
</ParamField>

<ParamField path="podTemplate" type="core/v1.PodTemplateSpec">
  Pod template for the component's pods. The operator injects its defaults (image, command, env, ports, probes, resources, volume mounts) into the container named `main`, merging your overrides by name. If no `main` container is present, the operator generates one. Every other container is treated as a user-managed sidecar and receives no injected defaults â€” sidecars must specify their own required fields such as `image`. Replaces the ten separate per-component fields (`resources`, `envs`, `livenessProbe`, and so on) that existed in `v1alpha1`.
</ParamField>

<ParamField path="multinode" type="object">
  Configures a multinode component. See [Multinode Deployments](deployment/multinode-deployment.md).
</ParamField>

<Indent>
  <ParamField path="multinode.nodeCount" type="integer" default="2">
    Number of nodes to deploy. Minimum `2`. Total GPUs used is `nodeCount Ă— container GPU request`.
  </ParamField>
</Indent>

<ParamField path="sharedMemorySize" type="quantity">
  Size of the tmpfs mounted at `/dev/shm`. Omit to use the operator default (`8Gi`); set a positive quantity for a custom size; set `"0"` to disable the shared-memory volume entirely.
</ParamField>

<ParamField path="globalDynamoNamespace" type="boolean" default="false">
  Places the component in the global Dynamo namespace rather than the per-deployment namespace derived from the DGD name.
</ParamField>

<ParamField path="modelRef" type="object">
  References a model served by this component. When set, a headless service is created for endpoint discovery.
</ParamField>

<Indent>
  <ParamField path="modelRef.name" type="string" required={true}>
    Base model identifier, for example `llama-3-70b-instruct-v1`.
  </ParamField>
  <ParamField path="modelRef.revision" type="string">
    Model revision or version.
  </ParamField>
</Indent>

<ParamField path="scalingAdapter" type="object">
  Opts the component into a [DynamoGraphDeploymentScalingAdapter](autoscaling.md) (DGDSA). Set it â€” even as an empty object, `scalingAdapter: {}` â€” to create a DGDSA that owns `replicas` so external autoscalers (HPA, KEDA, Planner) can drive scaling through the Scale subresource. Omit the field to opt out.
</ParamField>

<ParamField path="frontendSidecar" type="string">
  Designates a container in `podTemplate.spec.containers` as the frontend sidecar. The value must match a container `name` in that list; the operator merges its frontend-sidecar defaults (Dynamo env vars, ports, health probes) into that container the same way it merges into `main`. The validation webhook rejects values that match no container.
</ParamField>

<ParamField path="compilationCache" type="object">
  Configures a PVC-backed compilation cache. The operator handles backend-specific mount paths and environment variables, so you do not hand-wire them into `podTemplate`.
</ParamField>

<Indent>
  <ParamField path="compilationCache.pvcName" type="string" required={true}>
    Name of a user-created PVC, which must exist in the same namespace as the deployment.
  </ParamField>
  <ParamField path="compilationCache.mountPath" type="string">
    Overrides the backend-specific default mount path. When empty, the operator selects a default appropriate for the backend framework.
  </ParamField>
</Indent>

<ParamField path="eppConfig" type="object">
  EPP-specific configuration. Only valid when `type` is `epp`. Exactly one of `configMapRef` or `config` must be set. See [Gateway API Inference Extension](inference-gateway.md).
</ParamField>

<Indent>
  <ParamField path="eppConfig.configMapRef" type="core/v1.ConfigMapKeySelector">
    References a user-provided ConfigMap key containing EPP configuration. Mutually exclusive with `config`.
  </ParamField>
  <ParamField path="eppConfig.config" type="object">
    EPP `EndpointPickerConfig` supplied inline. The operator marshals it to YAML and creates the ConfigMap for you. Mutually exclusive with `configMapRef`.
  </ParamField>
</Indent>

<ParamField path="topologyConstraint" type="object">
  Component-level topology placement. See [Topology-Aware Scheduling](topology-aware-scheduling.md).
</ParamField>

<Indent>
  <ParamField path="topologyConstraint.packDomain" type="string" required={true}>
    Topology domain to pack pods within. Must match a domain defined in the referenced `ClusterTopology`. When the parent DGD also sets `spec.topologyConstraint.packDomain`, this value must be narrower than or equal to it.
  </ParamField>
</Indent>

<ParamField path="experimental" type="object">
  Opt-in preview features whose API shape may change in breaking ways between `v1beta1` releases. Fields here are **not** covered by the normal `v1beta1` deprecation policy â€” do not rely on them for production workloads. See [Experimental fields](#experimental-fields).
</ParamField>

## Experimental fields

<Warning>
  `spec.experimental` groups preview features that can change or disappear between `v1beta1` releases without a name-preserving graduation path. They are excluded from the `v1beta1` deprecation policy.
</Warning>

<ParamField path="experimental.gpuMemoryService" type="object">
  Configures the GPU Memory Service (GMS). When set, GPU access for GMS clients is managed through Dynamic Resource Allocation (DRA), and the operator replaces the `main` container's GPU resources with a DRA `ResourceClaim`. Key sub-fields: `mode` (`IntraPod` default, or `InterPod`), `deviceClassName` (default `gpu.nvidia.com`), and `extraClientContainers`.
</ParamField>

<ParamField path="experimental.failover" type="object">
  Configures active-passive GPU failover for a worker component. Requires `gpuMemoryService` to be set with a matching `mode`, and the `nvidia.com/dynamo-kube-discovery-mode: container` annotation on the DGD. See [Shadow Engine Failover](shadow-engine-failover.md).
</ParamField>

<ParamField path="experimental.checkpoint" type="object">
  Configures container-image snapshotting and restore for the component. Set `checkpoint.enabled: true` to opt in; omit `checkpointRef` for a DGD-managed automatic checkpoint, or set it to restore an existing [DynamoCheckpoint](snapshot.md). Sub-fields include `startupPolicy` (`Immediate` default, or `WaitForCheckpoint`), `deletionPolicy` (`Delete` default, or `Retain`), and `targetContainerName` (default `main`).
</ParamField>

## Status

The operator maintains observed state under `status`.

<ParamField path="status.conditions" type="[]metav1.Condition">
  Standard Kubernetes conditions. `Available` reports whether the component is serving traffic; `DynamoComponentReady` reports whether the underlying Dynamo component is ready.
</ParamField>

<ParamField path="status.observedGeneration" type="integer">
  Most recent `metadata.generation` the controller has reconciled. A component is up to date when this equals `metadata.generation`.
</ParamField>

<ParamField path="status.component" type="object">
  Replica status for this component (desired, ready, and available counts).
</ParamField>

## Inspect a DCD

```bash
# List component deployments (short name: dcd)
kubectl get dcd -n $NAMESPACE

# Detailed status and conditions
kubectl describe dcd my-worker -n $NAMESPACE

# Readiness at a glance
kubectl get dcd my-worker -n $NAMESPACE \
  -o jsonpath='{.status.conditions[?(@.type=="Available")].status}'
```
