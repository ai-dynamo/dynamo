Dynamo is an open-source, high-throughput, low-latency inference framework for serving generative AI workloads in distributed environments. It is Kubernetes-native for production while still supporting local containers, Python workers, and standalone components for development or incremental adoption.

New here? Pick an install path below. For the full story on Dynamo's design principles, performance techniques, and production-grade features, see [About Dynamo](about.md).

## Get Started

Dynamo is **Kubernetes-native, but not Kubernetes-only**. The full feature set — operator, CRDs, autoscaling, inference gateway, and multinode scheduling — runs on Kubernetes, while the same frontend/router/worker stack also runs locally for evaluation or incremental adoption. This user guide assumes you're running on a Kubernetes cluster, if you want to build from source or run in your local environment, see the developer's guide.
