# Figure tools — agentic-harnesses blog

Source for the three figures embedded in `../agentic-harnesses.md`. Each figure
is generated by a Plotly script that emits both a 2x PNG and an SVG into
`../images/`.

## Inventory

| # | Image | Generator | Subject |
|---|-------|-----------|---------|
| 1 | `fig-1-agent-loop.{png,svg}` | `gen_fig_1_agent_loop.py` | Standard server vs Dynamo across two turns of an agent loop. |
| 2 | `fig-2-ttft-prefix-stability.{png,svg}` | `gen_fig_2_ttft_prefix_stability.py` | TTFT for stable / varying / stripped prefixes on a 52K-token prompt. |
| 3 | `fig-3-streaming-dispatch-timeline.{png,svg}` | `gen_fig_3_streaming_dispatch.py` | Tool dispatch timing: standard server waits for end-of-stream; Dynamo emits `tool_call_dispatch` mid-stream. |

## Design system

`design_tokens.yaml`, `plotly_dynamo.py`, and `theme.d2` are copied verbatim
from `docs/blogs/flash-indexer/tools/`. Do not edit; if the upstream tokens
change, re-copy.

## Build

```bash
./build.sh
```

Requires `python3` with `plotly`, `kaleido`, `numpy`, and `pyyaml` available
on the path. Each script writes both PNG (2x) and SVG.

## Conventions

- Dynamo Dark palette; Geist + Geist Mono typography.
- "What Dynamo added" renders in `dynamo_green` (#76b900). Failure modes use
  `coral`. Neutral/baseline state uses `text.medium` gray.
- SVG is the embedded artifact in the blog. PNG is checked in alongside as a
  fallback for surfaces that do not render SVG.
