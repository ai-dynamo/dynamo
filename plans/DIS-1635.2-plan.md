# Plan: Reject duplicate model names on the same endpoint

## Context

Currently, two different models with different names can register at the same endpoint (namespace/component/endpoint). This causes undefined behavior at the frontend which expects a single model per endpoint. We need to add a check in the Rust discovery/registration path that rejects a model if a different model name is already registered at that endpoint.

Multiple workers serving the **same** model must still be allowed (they register with different instance_ids but the same model name). The race between concurrent registrations must be handled (check-then-act is not sufficient alone).

## Approach: Pre+post registration check in `lib/llm/`

Add a helper function in `lib/llm/src/discovery.rs` that wraps `discovery.register()` with validation. Both registration paths in `lib/bindings/python/rust/lib.rs` will use it.

### Step 1: Add validation helper in `lib/llm/src/discovery.rs`

Add a new public async function, e.g. `register_model_exclusive`:

```rust
pub async fn register_model_exclusive(
    discovery: &dyn Discovery,
    spec: DiscoverySpec,
) -> anyhow::Result<DiscoveryInstance>
```

Logic:
1. Extract the (namespace, component, endpoint) and display_name from the spec (the spec is `DiscoverySpec::Model` containing `card_json`).
2. **Pre-check**: Call `discovery.list(DiscoveryQuery::EndpointModels { namespace, component, endpoint })`. For each existing instance, deserialize the `card_json` and extract `display_name`. If any existing model has a different `display_name` than ours, return an error immediately.
3. **Register**: Call `discovery.register(spec)`.
4. **Post-check** (race detection): Call `discovery.list(EndpointModels {...})` again. Check all registered models. If there are now models with different display_names, unregister ourselves and return an error.

Error message should be clear, e.g.: `"Cannot register model '{}' on endpoint '{}/{}/{}': a different model '{}' is already registered there"`

Files to modify:
- `lib/llm/src/discovery.rs` — add `register_model_exclusive` function and the necessary imports

### Step 2: Call the helper from both registration paths in `lib/bindings/python/rust/lib.rs`

There are two places that call `discovery.register(spec)`:
1. **Line ~366** — tensor/images/videos fast path
2. **Line ~483** (in `local_model.rs:attach()`) — regular model path

For path 1 (`lib/bindings/python/rust/lib.rs:~366`): Replace `discovery.register(spec)` with `dynamo_llm::discovery::register_model_exclusive(discovery, spec)`.

For path 2 (`lib/llm/src/local_model.rs:~483`): Replace `discovery.register(spec)` with `crate::discovery::register_model_exclusive(discovery, spec)`.

### Step 3: Python side — minimal changes

The Python code already propagates exceptions. In `components/src/dynamo/sglang/register.py`, the `register_model()` call is wrapped in try/except that logs and returns False. `register_model_with_readiness_gate` already calls `engine.shutdown()` and raises `RuntimeError` on failure. This is sufficient — no Python changes needed.

Similarly for vllm/mocker backends — they already handle registration failures.
