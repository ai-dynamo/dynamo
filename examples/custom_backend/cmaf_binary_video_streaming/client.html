Dynamo CMAF Binary Stream POC

POST one prompt, then play a continuous CMAF/fMP4 stream over the experimental single-request binary route using Media Source Extensions.

This client expects `POST /v1/videos/stream/binary/cmaf` and uses `MediaSource` with one init fragment plus streamed `.m4s` media fragments. If the page and Dynamo are served from different origins, the videos router must expose CORS headers for browser testing.
