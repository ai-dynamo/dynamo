# kube-prometheus-stack - Production Addon

kube-prometheus-stack is the baseline metrics and alerting backbone. It installs Prometheus Operator, Prometheus, Alertmanager, Grafana, and the CRDs consumed by the other production add-ons.

## Deployment

| Field | Value |
|---|---|
| Argo CD app | [`gitops/apps/10-prometheus.yaml`](../../gitops/apps/10-prometheus.yaml) |
| Chart | `kube-prometheus-stack` |
| Source | `https://prometheus-community.github.io/helm-charts` |
| Version | `84.1.0` |
| Namespace | `monitoring` |
| Values | [`values.yaml`](values.yaml) |
| Sync wave | `1` |
| Baseline | Yes |

## What This Addon Owns

| Capability | Local setting |
|---|---|
| Stable release name | `fullnameOverride: kube-prometheus-stack`. |
| Grafana | Enabled. |
| Grafana dashboard discovery | Sidecar dashboards enabled, `searchNamespace: ALL`. |
| Grafana datasource discovery | Sidecar datasources enabled. |
| Prometheus retention | `15d`. |
| Cross-namespace scraping | `ServiceMonitor`, `PodMonitor`, and rule namespace selectors are open. |
| Alertmanager | Enabled. |

## What This Addon Does Not Own

- It does not create every `ServiceMonitor`; individual add-ons create their own.
- It does not own application log storage; Loki does.
- It does not own traces; OpenTelemetry does when enabled.

## Argo Drift Rules

The Argo CD app ignores Grafana's generated admin password and the deployment checksum derived from it. Those fields are generated by the chart/runtime and should not keep the production app OutOfSync.

## Dynamo Contract

Other add-ons label their ServiceMonitors with `release: kube-prometheus-stack`. Keep that release label stable unless every dependent values file is updated at the same time.

## Verify

```bash
kubectl -n monitoring get prometheus,alertmanager,pods
kubectl get servicemonitor -A
kubectl get prometheusrule -A
deploy/pre-deployment/pre-deployment-check.sh --require prometheus-operator,prometheus-instance,prometheus-rules
```

## Upgrade Notes

kube-prometheus-stack upgrades can update Prometheus Operator CRDs. Review CRD changes and confirm custom `ServiceMonitor` and `PrometheusRule` resources still reconcile.

Upstream reference: [Prometheus community Helm chart](https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-prometheus-stack).
